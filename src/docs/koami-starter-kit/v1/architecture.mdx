# Architecture

Understanding how KoamiStarterKit is organized.

## Project Structure

```
my-app/
├── app/
│   ├── Actions/          # Fortify actions (auth logic)
│   ├── Console/          # Artisan commands
│   ├── Filament/         # Admin panel resources
│   ├── Http/
│   │   ├── Controllers/  # Web controllers
│   │   └── Middleware/   # Inertia, appearance, etc.
│   └── Models/           # Eloquent models
├── resources/
│   ├── js/               # Vue 3 frontend
│   │   ├── components/   # Vue components + Shadcn UI
│   │   ├── layouts/      # Page layouts
│   │   ├── pages/        # Inertia page components
│   │   └── wayfinder/    # Auto-generated route types
│   └── views/            # Blade templates
├── routes/
│   ├── web.php           # Application routes
│   └── auth.php          # Fortify auth routes
└── tests/                # Pest tests
```

## Key Concepts

### Hybrid Frontend

Two separate frontends share the same backend:

| Frontend | Technology | Users | Location |
|----------|-----------|-------|----------|
| Public | Vue 3 + Inertia | Regular users | `resources/js/pages` |
| Admin | Filament (Blade/Livewire) | Admins only | `app/Filament` |

### Wayfinder (Type-Safe Routes)

Wayfinder scans your Laravel routes and generates TypeScript definitions:

```typescript
// Auto-generated - don't edit
import { route } from '@/wayfinder'

// Full type safety + autocomplete
route('users.show', { user: 1 })  // ✓ knows 'user' is required
route('users.index')              // ✓ no params needed
```

### Authentication Flow

1. **Request** → Laravel Fortify handles validation
2. **Session** → Authenticated via session cookie
3. **Inertia** → Renders Vue components server-side
4. **SPA feel** → Subsequent navigation is client-side

## Security

- **Auth**: Laravel Fortify (battle-tested)
- **Roles**: Spatie Permissions (Admins vs Users)
- **Impersonation**: Super-admins can login as other users (with visual banner)
- **CSRF**: Automatic protection on all forms
- **XSS**: Vue escapes HTML by default
